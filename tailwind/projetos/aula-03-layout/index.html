Display Utilities

O Tailwind traduz cada comportamento de display em classes curtas. Nesta aula, mapeamos cada tipo de comportamento para o CSS tradicional.

Display

Block

A classe block força o elemento a preencher a largura disponível e quebrar linha antes e depois.

<div class="block w-full">...
Conteúdo
</div>

Equivalente CSS: display: block;

Analogia: uma pessoa com os braços abertos que ocupa toda a largura da fila.

Display

Inline

inline deixa o elemento fluir ao lado dos outros e limita largura ao conteúdo.

<span class="inline">Link</span>

Equivalente CSS: display: inline;

Analogia: alguém normal na fila do banco, ocupando só o espaço que precisa.

Display

Inline-block

inline-block combina o melhor do block e do inline: respeita largura e altura, mas não quebra linha automaticamente.

<div class="inline-block w-32 h-20">Caixa</div>

Equivalente CSS: display: inline-block;

Analogia: uma pessoa que aceita largura e altura, mas ainda pode ficar ao lado de outras.

Display

Flex

flex ativa o modo Flexbox e transforma os filhos em flex items com controle fino de direções, alinhamento e distribuição.

<div class="flex gap-4">
<div>Item 1</div>
<div>Item 2</div>
</div>

Equivalente CSS: display: flex;

Analogia: um organizador de fila que decide direção, distribuição e alinhamento.

Display

Grid

grid ativa o layout bidimensional. Combinamos posteriormente colunas, linhas e gaps.

<div class="grid grid-cols-3 gap-4">
<div>Celula</div>
</div>
</div>

Equivalente CSS: display: grid;

Nota: aprofundamos Grid na próxima aula, mas já entendemos o comando base.

Display

Hidden

hidden remove o elemento do fluxo, assim como display: none.

<div class="hidden">Escondido</div>

Equivalente CSS: display: none;

Lembre: invisible mantém o espaço e opacity-0 apenas tira a opacidade.

A chave para dominar layout é mapear mentalmente cada classe para o CSS equivalente. Lembre-se: block é a base, inline é leve, e flex/grid são ferramentas poderosas para organizar filas e quadros.

Position Utilities e Camadas

Agora você controla a posição dos elementos. As classes position, top/right e z-index são os instrumentos principais para compor layouts dinâmicos.

Static & Relative

static é o padrão: o elemento fica exatamente onde está no fluxo. relative permite deslocamentos sem remover o espaço dele do layout.

<div class="relative top-4 left-4">Desliza sem quebrar o fluxo</div>

Equivalente CSS: position: relative; top: 1rem; left: 1rem;

Use relative sempre que quiser servir de referência para elementos absolute.

Absolute, Fixed e Sticky

absolute remove o elemento do fluxo e posiciona em relação ao ancestral com position diferente de static. fixed trava o componente na viewport e sticky "gruda" depois de rolar.

  • Absolute: use top-4 right-4 para encaixar cantos.
  • Fixed: ótimo para headers persistentes; combine w-full e top-0.
  • Sticky: mescla relative + fixed — o elemento permanece no fluxo até atingir o limite.

Top, Right, Bottom, Left e Insets

As classes top-{valor}, right-{valor} etc. recebem números, frações e negativos. Use também inset-0, inset-x-0 e inset-y-0 para economizar digitação.

<div class="relative h-52 bg-slate-100">
<div class="absolute inset-x-0 top-4 mx-auto w-48">Caixa centralizada</div>
</div>

Overlay + modal

Combine fixed, inset-0 e z-40 para estruturar modais com overlay.

<div class="fixed inset-0 bg-black/50 z-40">
<div class="absolute top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2">Modal</div>
</div>

Flexbox: organizando a festa

Pense no Flexbox como o organizador de convidados: ele define direção, alinhamento, distribuição e espaçamento. Com flex, o Tailwind dá comandos precisos para você falar com o layout.

Direções: Row e Column

flex-row empilha itens em linha, flex-col em coluna. Use o mobile-first para alternar facilmente entre os dois.

Row Row Row
Col Col Col

Distribuição e alinhamento

justify-* controla o eixo principal, items-* o eixo cruzado.

justify-between itens distribuídos
justify-center
items-start

Gap e Wrap: espaçamento automático

gap-* distribui espaçamento entre os itens automaticamente. flex-wrap permite que os itens quebrem linha quando necessário.

Centralização completa

flex items-center justify-center h-screen é o truque mais rápido para colocar algo exatamente no meio da tela.

<div class="flex items-center justify-center h-screen">
<div>Centralizado</div>
</div>

Flex-grow, flex-shrink e flex-1

flex-1 expande o item para preencher o espaço restante. flex-grow e flex-shrink ajustam como o item cresce e encolhe.

flex-1
flex-[2]
flex-[0.5]

Combine flex-1 com min-w-0 para evitar overflow em colunas e use flex-shrink-0 quando não quiser que o item encolha.

Exercícios práticos

Coloque as mãos na massa com os desafios principais da aula e consolide o raciocínio sobre layout.

Exercício 1 · Header responsivo

Recrie um header usando Flexbox: logo à esquerda, menu ao centro, botão à direita, padding 1rem, altura 4rem e sombra sutil.

  • Flexbox horizontal (logo, nav, botão).
  • Centralize o menu e mantenha o botão destacado.
  • Use shadow e padding uniforme.

Exercício 2 · Card com layout flex

Card vertical com imagem, título, descrição e rodapé usando flex-col.

  • Imagem com altura 200px.
  • Rodapé com preço à esquerda e botão à direita.
  • Gap de 1rem entre os blocos.

Exercício 3 · Modal centralizado

Crie overlay escuro e modal centralizado com botão de fechar posicionado com absolute.

  • Overlay com fixed inset-0.
  • Modal width 400px, padding 2rem e border radius.
  • Botão absoluto no canto superior direito.

Exercício 4 · Sidebar fixa

Layout com sidebar à esquerda (256px) e conteúdo principal à direita, ocupando 100vh.

  • Sidebar com background escuro e texto branco.
  • Menu vertical com links.
  • Conteúdo com padding 2rem.

Exercício 5 · Formulário flexível

Formulário em coluna com espaço de 1rem entre os campos e botões alinhados lado a lado com largura igual.

  • Campos com label acima do input.
  • flex gap-4 nos botões.
  • Centralize o formulário em telas grandes.

Exercício 6 · Header sticky

Header com sticky top-0, sombra e z-index alto para permanecer no topo durante o scroll.

  • Use shadow-lg apenas quando sticky.
  • Conteúdo suficiente abaixo para gerar rolagem.
  • Menu centralizado com logo e botão lateral.

Exercício 7 · Grid de cards com wrap

Container flex com wrap, cada card com 300px de largura e gap 1.5rem.

  • Use flex flex-wrap gap-6.
  • Cards com imagem, título, descrição e botão.
  • Cards centralizados quando não preenchem a linha.

Exercício 8 · Analise o layout

Identifique problemas nos três blocos de código e corrija layout, alinhamento e prioridade dos itens.

  • Adicione flex-1 ao conteúdo principal.
  • Uso de items-center para centralizar quadrados.
  • justify-center faltante no terceiro exemplo.

Conexão com o CSS tradicional

Nunca perca de vista o que cada classe representa em CSS. Use estas referências sempre que estiver em dúvida.

Layout

flex → display: flex
grid → display: grid
hidden → display: none

Alinhamento & espaçamento

justify-center → justify-content: center
items-center → align-items: center
gap-4 → gap: 1rem

Position

relative → position: relative
absolute top-4 right-6 → top: 1rem; right: 1.5rem
inset-0 → top: 0; right: 0; bottom: 0; left: 0

Camadas e prioridades

z-10 → z-index: 10
fixed top-0 → trava no viewport acima de tudo
