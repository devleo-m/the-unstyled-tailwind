Aula 5 · Responsividade com Tailwind CSS

Nesta aula explicamos como o Tailwind transforma media queries em prefixos claros, mantendo o foco em mobile-first e tornando cada ajuste previsível.

mobile-first breakpoints prefixos container

Cada seção abaixo ensina com analogias simples, exemplos comparando Tailwind e CSS puro, e códigos prontos para seguir como professor da própria jornada.

Revisão

O que é responsividade?

Responsividade significa fazer o site responder ao tamanho da tela. Em CSS puro você encaixa media queries, no Tailwind basta adicionar prefixos como `md:` e `lg:`.

CSS Puro

Você escreve media queries com `min-width` ou `max-width` e controla padding, fontes e layout manualmente.

/* Mobile first */
.container { width: 100%; padding: 1rem; }
@media (min-width: 768px) { .container { padding: 2rem; } }
@media (min-width: 1024px) { .container { padding: 3rem; } }

Tailwind

O mesmo efeito acontece adicionando classes com prefixos: o estilo sem prefixo é mobile, e `md:`/`lg:` ativam estilos maiores.

<div class="p-4 md:p-8 lg:p-12">Conteúdo</div>

Filosofia

Mobile-First em ação

Por que começar pequeno?

  • Mobile domina o acesso hoje.
  • Performance melhora se você não carrega estilos pesados desnecessários.
  • Adicionar complexidade gradualmente evita confusão.

Prefixos responsivos

Use `[breakpoint]:[classe]`. O Tailwind aplica em cascata: primeiro mobile (sem prefixo), depois `sm`, `md`, `lg`, `xl` e `2xl`.

<div class="text-sm md:text-base lg:text-lg">Texto responsivo</div>

Ordem importa: sempre comece com o estilo mobile e siga os breakpoints em ordem crescente para que o Tailwind sobreponha corretamente.

Breakpoints

Padrões prontos para usar

Prefixo Tamanho Uso comum
(sem prefixo) < 640px Mobile
sm: ≥ 640px Smartphones grandes
md: ≥ 768px Tablets
lg: ≥ 1024px Desktops
xl: ≥ 1280px Desktops grandes
2xl: ≥ 1536px Telas muito grandes

Os valores vêm do `tailwind.config.js`. Se você abrir, verá `screens: { sm: '640px', md: '768px', ... }`. Pode estender ou substituir esses valores, dependendo da necessidade.

// tailwind.config.js
screens: {'{'}
'sm': '640px',
'md': '768px',
'lg': '1024px',
'xl': '1280px',
'2xl': '1536px',
{'}'}

Responsividade prática

Ajustando propriedades chave

Padding, margem, tipografia, layout e cores ganham prefixos que escalam com a tela.

Espaçamento responsivo

Use `sm:`, `md:` etc. em `p-`, `m-` e `gap-` para manipular cartões, grids e stacks.

<div class="p-2 md:p-6 lg:p-8 gap-2 md:gap-4 lg:gap-6">...

Tipografia responsiva

Tamanhos, peso e altura mudam de acordo com o breakpoint.

<h1 class="text-2xl sm:text-3xl md:text-4xl lg:text-5xl">Título</h1>
<p class="text-sm ld:text-base md:text-lg leading-relaxed">Parágrafo</p>

Layout responsivo

Transforme `flex` e `grid` com prefixos para reorganizar pilhas e colunas.

<div class="flex flex-col md:flex-row lg:grid lg:grid-cols-3">

Cores e visibilidade

Mude backgrounds, opacidade e visibilidade conforme o breakpoint para destacar ou esconder elementos.

<div class="bg-blue-500 md:bg-green-500 lg:bg-purple-500 hidden md:block">Banner</div>

Lembre-se: `md:hidden` e `md:block` ativam ou escondem elementos e são ótimos para alternar menus e controles.

Customização

Breakpoints e container personalizados

Se o projeto exige valores diferentes, você pode estender os `screens` no `tailwind.config.js` ou redefinir a classe `.container` para centralizar e aplicar padding automático.

module.exports = {'{'}
theme: {'{'}
extend: {'{'}
screens: {'{'}
'xs': '475px',
'3xl': '1920px',
{'}'}
{'}'}
{'}'}
{'}'}

Depois, use `xs:p-6` ou `3xl:px-12` para aplicar novas paradas. A classe `.container` também respeita esses breakpoints e pode ganhar `padding` extra.

Variantes responsivas

Combinando hover, focus e prefixos

<button class="bg-blue-500 hover:bg-blue-600 md:hover:bg-blue-700">CTA</button>
<input class="border focus:border-blue-500 md:focus:ring-2" />

Você combina qualquer variante (`hover:`, `focus:`, `disabled:`) com breakpoints e o Tailwind resolve a ordem automaticamente.

Debug

Como testar responsividade

Abra o DevTools, ative o modo responsivo (Cmd+Shift+M) e simule diferentes breakpoints. Observe as classes aplicadas para entender o que muda.

Você também pode inspecionar o CSS gerado para ver as media queries equivalentes e confirmar se os prefixos seguem a ordem correta.

@media (min-width: 768px) {'{'}
.md\:p-8 {'{'} padding: 2rem; {'}'}

Exercícios & reflexões

Pratique e pense como professor

Card de produto responsivo

Padding, fontes e layout ajustam com `sm:`, `md:` e `lg:`. Botão e preço mudam de coluna para linha.

  • Use `p-4 md:p-6 lg:p-8`, `text-lg md:text-xl lg:text-2xl` e `h-32 md:h-48 lg:h-64`.
  • Combine `flex flex-col sm:flex-row` para o bloco de preço.
  • Implemente `bg-white rounded-lg shadow-md` para o visual.

Grid de cards

Quantas colunas devem aparecer em cada breakpoint? Comece com 1 e vá até 4 no `xl`.

  • `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4`
  • `gap-4 md:gap-6 lg:gap-8` para respirabilidade
  • Cards usam `bg-white p-4 md:p-6 rounded-lg shadow`.

Navegação responsiva

Hamburger aparece só em mobile e menu horizontal só em desktop.

  • `hidden md:flex` para o menu e `block md:hidden` para o botão.
  • Use `bg-blue-900 text-white` e `hover:text-blue-200`.
  • Container com `flex justify-between items-center`.

Hero responsiva

Título, subtítulo e botão crescem e o padding aumenta.

  • `text-3xl md:text-5xl lg:text-7xl` para o título
  • `py-8 md:py-16 lg:py-24` para o espaçamento vertical
  • Botão com `px-4 py-2 md:px-6 md:py-3 lg:px-8 lg:py-4`

Reflexão para o quadro

Que decisões de breakpoint são essenciais para o projeto? Quando você precisa extender os valores padrão? Como garantir que o CSS gerado continue enxuto?

Resumo

O que ficamos

  • Tailwind usa mobile-first: tudo começa sem prefixo e evolui com `sm:`, `md:`, `lg:`.
  • Breakpoints padrão cobrem quase todo o espectro, mas podem ser estendidos no `tailwind.config.js`.
  • Espaçamento, tipografia, grid e cores respondem com prefixos e mantêm o HTML legível.
  • Combine variantes (`hover`, `focus`) com prefixos para controles responsivos.
  • Use DevTools para inspecionar quais classes estão sendo aplicadas em cada breakpoint.

Conexão com CSS puro

Cada classe tem um par no CSS

Padding & Gap

`md:p-8` = `@media (min-width: 768px) { padding: 2rem; }`

`lg:gap-6` = `@media (min-width: 1024px) { gap: 1.5rem; }`

Tipografia

`sm:text-base` = `@media (min-width: 640px) { font-size: 1rem; }`

`lg:text-3xl` = `@media (min-width: 1024px) { font-size: 1.875rem; }`

Visibilidade

`md:hidden` = `@media (min-width: 768px) { display: none; }`

`md:block` = `@media (min-width: 768px) { display: block; }`
